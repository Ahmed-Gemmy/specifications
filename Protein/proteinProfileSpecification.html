Detailed description

This Protein profile specification presents the BioChemEntity usage when describing a Protein.
The recommended type used in Bioschemas is PR:000000001

profile bioschemas:associatedTo MedicalContidion
URL
PropertyValue
 Schema:

Bioschemas protein profile: Used as short and direct alternative to additionalProperty. It represents the association between a protein and a disease. The disease should be modeled as a schema:MedicalCondition. Whenever possible, use the MedicalCode to refer to an ontology term such as those in OMIM. If PropertyValue is used as the object, use the property "value" for the medical condition and the propererty "hasEvidence" for any evidence related to the association. 		MANY Property corresponding to sio:SIO_000001 Association to a medical condition 
{
    "@type": ["BioChemEntity", "Protein"],
    "@id": "http://purl.uniprot.org/uniprot/P05519"
    "associatedTo": {
        "@type": "MedicalCondition",
        "additionalType": "http://semanticscience.org/resource/SIO_010299",
        "name": "Leukemia, chronic myeloid (CML)",
        "code": { "@type": "MedicalCode", "code": "608232", "codingSystem": "OMIM“ },
        "sameAs": "http://www.uniprot.org/diseases/DI-03735"
    }
}
Association to a medical condition with evidence 
{
    "@type": ["BioChemEntity", "Protein"],
    "@id": "http://purl.uniprot.org/uniprot/P05519"
    "associatedTo": {
        "@type": "PropertyValue",
        "value": {
            "@type": "MedicalCondition",
            "additionalType": "http://semanticscience.org/resource/SIO_010299",
            "name": "Leukemia, chronic myeloid (CML)",
            "code": { "@type": "MedicalCode", "code": "608232", "codingSystem": "OMIM“ },
            "sameAs": "http://www.uniprot.org/diseases/DI-03735"
        },
        "hasEvidence": "https://www.ncbi.nlm.nih.gov/pubmed/15201367"
    }
}

bioschemas.org isContainedIn BioChemEntity
URL
 Schema: Indicates a BioChemEntity that this BioChemEntity is (in some sense) part of.

Bioschemas: 		MANY Multiple options depending on the BioChemEntity. For instance, you can use NCBITaxon for organisms. If a protein belongs to an organism, it is recommended to model it whenever possible, make sure you link to a well-known taxonomy term: 
{
    "@type": ["BioChemEntity", "Protein"],
    "@id": "http://purl.uniprot.org/uniprot/P05519"
    "isContainedIn": {
        "@type": "BioChemEntity",
        "additionalType": "http://purl.obolibrary.org/obo/OBI_0100026",
        "identifier": "9606", "name": "Homo sapiens",
        "url": "http://purl.bioontology.org/ontology/NCBITAXON/9606",
        "sameAs": "http://purl.uniprot.org/taxonomy/9606"
    }
}
You can also model how a protein is contained in a particular orga: 
{
    "@type": ["BioChemEntity", "Protein"],
    "@id": "http://purl.uniprot.org/uniprot/P05519"
    "isContainedIn": {
        "@type": "BioChemEntity",
        "name": "Liver"
    }
}

schema.org additionalProperty PropertyValue
 Schema: A property-value pair representing an additional characteristics of the entitity, e.g. a product feature or another characteristic for which there is no matching property in schema.org. Note: Publishers should be aware that applications designed to use specific schema.org properties (e.g. http://schema.org/width, http://schema.org/color, http://schema.org/gtin13, ...) will typically expect such data to be provided using those properties, rather than using the generic property/value mechanism.

Bioschemas: As much a possible, do not use it! Alternatively, considered reusing properties/relations already coined in controlled vocabularies. We recommed to look at the OBO Relation Ontology (RO) or the Semanticscience Integrated Ontology (SIO) as starting points.

Bioschemas protein profile: Find a suitable relation or property already existing. As an example, take a look to bioschemas:transcribedFrom or bioschemas:associatedTo used in this ProteinProfile. 		MANY Any ontology term coined as a property and suitable for your needs. For instance sio:SIO_000095 (is member of) could be used to model the relation between a protein and a protein clan. Short direct alternative to express relation between protein and protein clan. 
{
    "@type": ["BioChemEntity", "Protein"],
    "@id": "http://purl.uniprot.org/uniprot/P05519"
    "http://semanticscience.org/resource/SIO_000095":"http://pfam.xfam.org/clan/CL0001"
}
If no ontology term expressing the desired relation exists, then 
{
    "@type": ["BioChemEntity", "Protein"],
    "@id": "http://purl.uniprot.org/uniprot/P05519"
    "additionalProperty": {
        "name": "my new property",
        "value": {
            "@type": "PropertyValue",
            "value": "the object in this relation, more properties can be used as needed"
        }
    }
}

URL
 Schema: Indicates a BioChemEntity that is (in some sense) a part of this BioChemEntity. Inverse property: isContainedIn.

Bioschemas: 		MANY A protein can contain a variant:

    "@id": "http://purl.uniprot.org/uniprot/P05519"

More information about the variant can also be included

        "@type": "BioChemEntity",

        "aditionalType": "http://semanticscience.org/resource/SIO_010471",
        "identifier": "VAR_022315",
        "location": [
            {
              "@type": "PropertyValue",
              "value": 355
            },
    }

bioschemas.org hasRepresentation PropertyValue
Text
URL
 Schema: A representation for this entity other than, for instance, an image (used image property for that) or the main web page/record (use mainEntityOfPage for that, and see background notes, http://schema.org/docs/datamodel.html#mainEntityBackground, for sameAs and url). Protein profile bioschemas:enables CategoryCode
PropertyValue
 Schema:

Bioschemas protein profile: A protein enables GO molecular functions. If you want to add an evidence use PropertyValue with "value" pointing to the CategoryCode corresponding to the GO term and "hasEvidence" to the evidence. Otherwise just use the GO term as the CategoryCode. 		MANY Gene Ontology (GO) Protein enabling a GO molecular function 
{
    "@type": ["BioChemEntity", "Protein"],
    "@id": "http://purl.uniprot.org/uniprot/P05519"
    "enables": {
        "@type": "CategoryCode",
        "@id": "http://purl.obolibrary.org/obo/GO_0000166",
        "codeValue": "GO:0000166",
        "name": "nucleotide binding"
    }
}
Evidence can also be modeled 
{
    "@type": ["BioChemEntity", "Protein"],
    "@id": "http://purl.uniprot.org/uniprot/P05519"
    "enables": {
        "@type": "PropertyValue",
        "value": {
            "@type": "CategoryCode",
            "@id": "http://purl.obolibrary.org/obo/GO_0000166",
            "codeValue": "GO:0000166",
            "name": "nucleotide binding"
        },
        "hasEvidence": {
            "@type": "CategoryCode",
            "@id": "http://purl.obolibrary.org/obo/ECO_0000256",
            "codeValue": "ECO:0000256",
            "name": "match to sequence model evidence used in automatic assertion"
        }
    }
}
